JavaScriptやTypeScriptの\*\*分割代入（Destructuring assignment）\*\*ですね。オブジェクトと配列では、データの取り出し方のルールが根本的に異なります。

一言で言うと、**「名前で探すのがオブジェクト」**、\*\*「順番で決めるのが配列」\*\*です。

## ---

**1\. オブジェクトの分割代入**

オブジェクトの場合、受け側の変数名は\*\*「元のプロパティ名」と同じ\*\*にする必要があります。

JavaScript

const user \= { id: 1, name: "Tanaka" };

// プロパティ名（id, name）と同じ名前で受ける  
const { id, name } \= user;

console.log(id);   // 1  
console.log(name); // "Tanaka"

### **別の名前で受け取りたいとき**

もし、違う変数名に格納したい場合は、元の名前: 新しい名前 という書き方をします。

JavaScript

const { name: userName } \= user;  
console.log(userName); // "Tanaka"

## ---

**2\. 配列の分割代入**

配列の場合、名前という概念がないため、**「書いた順番」に値が割り当てられます。受け側の名前は何でも自由に決めてOK**です。

JavaScript

const colors \= \["red", "blue", "green"\];

// 1番目、2番目の順に好きな名前をつける  
const \[firstColor, secondColor\] \= colors;

console.log(firstColor);  // "red"  
console.log(secondColor); // "blue"

### **特定の順番を飛ばしたいとき**

カンマを空けることで、特定の要素を無視できます。

JavaScript

// 1番目と3番目だけ欲しい場合  
const \[first, , third\] \= colors;

## ---

**違いのまとめ**

| 特徴 | オブジェクト { } | 配列 \[ \] |
| :---- | :---- | :---- |
| **識別のルール** | **プロパティ名**（Key） | **インデックス**（順番） |
| **受け側の変数名** | **元の名前と一致**させる必要がある | **自由**に決めて良い |
| **主な用途** | 必要なデータだけをピンポイントで抜く | リストの先頭から順に処理する |

### ---

**💡 迷った時のポイント**

* \*\*APIのレスポンスなど（{ id: 1, ... }）\*\*を受ける時は、中身の名前を確認して { } を使う。  
* \*\*Reactの useState など（\[value, setValue\]）\*\*を受ける時は、自分の好きな名前をつけて \[ \] を使う。
