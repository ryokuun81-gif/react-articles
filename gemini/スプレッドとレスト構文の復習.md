
##ちょっとスプレッド構文とレスト構文を忘れたので教えてもらえますか。

スプレッド構文（Spread Syntax）とレスト構文（Rest Parameters）、名前も見た目（...）もそっくりなので、こんがらがりますよね。

一言で言うと、**「バラバラにする」のがスプレッド**、**「一つにまとめる」のがレスト**です。

## ---

**1\. スプレッド構文（バラバラにする）**

配列やオブジェクトの中身を、外側の枠から解き放って**展開**するイメージです。

### **配列のコピーや結合**

JavaScript

const fruits \= \['apple', 'banana'\];  
const moreFruits \= \[...fruits, 'cherry', 'orange'\]; 

console.log(moreFruits); // \['apple', 'banana', 'cherry', 'orange'\]

### **オブジェクトの複製**

JavaScript

const user \= { name: 'Gemini', age: 1 };  
const updatedUser \= { ...user, location: 'Cloud' };

console.log(updatedUser); // { name: 'Gemini', age: 1, location: 'Cloud' }

## ---

**2\. レスト構文（一つにまとめる）**

複数の値を**残さず（Rest）一つの配列にギュッとまとめる**イメージです。関数の引数や、分割代入でよく使われます。

### **関数の引数**

引数の数が決まっていない時に便利です。

JavaScript

function sum(...numbers) {  
  // numbersは \[1, 2, 3\] という配列になる  
  return numbers.reduce((acc, curr) \=\> acc \+ curr);  
}

console.log(sum(1, 2, 3)); // 6

### **分割代入での活用**

「特定の項目以外を全部まとめたい」という時に使います。

JavaScript

const colors \= \['red', 'blue', 'green', 'yellow'\];  
const \[first, ...others\] \= colors;

console.log(first);  // 'red'  
console.log(others); // \['blue', 'green', 'yellow'\] (残りがまとまった！)

## ---

**どっちがどっち？の見分け方**

使いどころで判断するのが一番簡単です！

* **右辺（値として使うとき）にある ...**  
  * const a \= \[...b\] → 中身を広げているので **スプレッド**  
* **左辺（受け皿として使うとき）にある ...**  
  * function(arg, ...rest) や const \[x, ...y\] \= arr → 余りをまとめているので **レスト**

\[\!TIP\]

\*\*「広げるスプレッド、まとめるレスト」\*\*とリズムで覚えると忘れにくいですよ。

##...fruits,でスプレッド構文だと外側の配列もなくなって単体になる感じですかね。

その通りです！まさに\*\*「外側の殻をパカッと割って、中身をぶちまける」\*\*というイメージが完璧に正解です。

スプレッド（Spread）という言葉自体に「広げる」「散布する」という意味があるように、...fruits と書いた瞬間に、それは配列ではなく\*\*「バラバラの要素」\*\*になります。

### ---

**イメージ図**

イメージとしては、こんな魔法がかかっている状態です。

JavaScript

const fruits \= \['apple', 'banana'\];

// イメージ：\[...fruits\]   
// 1\. 中身を取り出す： 'apple', 'banana'  
// 2\. 新しい皿 \[\] に並べる： \['apple', 'banana'\]

もし、... を使わずに \[fruits\] と書くと、\*\*「箱の中に箱が入っている」\*\*状態（二次元配列）になってしまいます。

JavaScript

const fruits \= \['apple', 'banana'\];

const spread \= \[...fruits\]; // 結果: \['apple', 'banana'\] （中身だけ移動）  
const noSpread \= \[fruits\];   // 結果: \[\['apple', 'banana'\]\] （箱ごと移動）

### **なぜ「単体」にする必要があるのか？**

一番のメリットは、\*\*「既存の配列を壊さずに、新しい配列を作れること」\*\*です。

JavaScript

const original \= \[1, 2\];  
const copy \= \[...original, 3\]; // originalはそのまま \[1, 2\]、copyは \[1, 2, 3\]

これを使わずに copy \= original としてしまうと、JavaScriptの性質上、片方を変えるともう片方も変わってしまう（参照渡し）ので、スプレッド構文で「バラバラにしてから詰め直す」という作業がとても重要になります。

### ---

**ちょっとした応用：関数の引数**

「単体になる」という感覚を一番実感できるのは、Math.max などを使う時かもしれません。

JavaScript

const prices \= \[100, 500, 300\];

// Math.max(prices) と書くと「配列は比較できない！」と怒られますが…  
// スプレッドでバラせばOK！  
console.log(Math.max(...prices)); // Math.max(100, 500, 300\) と同じ意味になる
